Declarative Design

Jeremy Keith - 12-03-2025

Programmeren += Muziek

Je kan net als klassieke muziek alles letterlijk opschrijven en zeggen welke noot wanneer moet, maar je kan het ook net als jazz een soort guidelines geven. 🎹 🎺

In programmeren werkt het hetzelfde, je hebt 2 soorten programmering:

→ Imperative programming
Je kan stap voor stap dingen opschrijven, de meeste programma’s werken zo.

→ Declarative programming
Hier kan je dingen schrijven en onderweg dingen veranderen.

HTML → Declarative

CSS → Declarative

“Every line of CSS is a suggestion for your website”

JS → Imperative
Alles gaat stuk als je een error hebt.

Controle over je code

Net als bij coderen heb je 2 soorten mindset.
Je hebt de makkelijke manier, over hoe het eruitziet, en hoe het werkt.

→ Waarom je dit zou doen, komt eigenlijk alleen maar op controle neer.
Hoe minder je schrijft, hoe minder je controle hebt over de site.

“JavaScript should only do what only JavaScript can do”

Je kan zelf kiezen hoeveel controle je wilt; heel basis:

body {
  font-size: 16px;
  padding-left: 16px;
}

Of heel uitgebreid:

body {
  font-size: clamp(1rem, 0.5rem + 0.666vw, 1.5rem);
  padding-inline-start: 1rem;
}

Bepaal zelf de min en max, en laat de computer de rest uitrekenen.

“Be the browser’s mentor, not its micromanager”

Declarative > Imperative? It depends

Het gaat om de werkcultuur waar je in werkt. Dit kan veel variabelen zijn en hangt vaak van het bedrijf of land af.

De werkcultuur ligt nooit echt vast, het is meer een sfeer binnen het bedrijf.

D.m.v. een design system kan je binnen een bedrijf vaak vastleggen hoe de cultuur werkt.

Thinking about thinking

  • Analytic thinking
    Heel goed voor wetenschappelijk en specifiek werk.
  • Systems thinking
    Het snappen van het concept, en niet per se in het detail treden ervan.

Je kan letterlijk de kleurcodes geven van de knop, of je kan zeggen dat de schaduw 10% lichter kan zijn dan de achtergrond. Het ligt aan het team waar je in werkt en hoe zij ermee omgaan.



Dus welke is nou het beste? It depends ✨
