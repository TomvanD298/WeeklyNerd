Pay the Web Forward with web Monetization

Peter-Paul Koch - 26-02-25

Waarom zouden websites gratis moeten zijn?

Advertenties zijn eigenlijk een soort schaamte voor mensen zodat ze niet hoeven te betalen voor het gebruik van websites. Dit komt omdat mensen eigenlijk er niet voor willen betalen, en niemand weet waarom. Het is een slechte gewoonte en dit moeten we afleren???

Als je veel geld wilt verdienen moet je een hype creëren. De meeste Hypes zijn dingen die trendy zijn op dit moment.

AI is nu een trend, over 3 jaar is Slubdiewub dat.

Wat willen we?

“Je moet betalen of anders”

Dit hoeft natuurlijk niet. Maar het zou beter werken als je een collectebus had. Bijvoorbeeld 5 cent per bezoek van een website.

Dit geldt natuurlijk voor websites met blogs, nieuws etc., niet bij webshops.

Hoe leuker je de website vindt, hoe meer je betaalt — dit kan je meten door tijd.

Web Monetization

Dit kan worden opgelost door Web Monetization. Bijvoorbeeld je hebt 10 euro, en elke keer als je op de website komt spendeer je 5 cent.

Hoe gaat dit gebeuren?

Hoe kan je met web monetization betalen?

  • Wallet — zoals een bankrekening.
  • Een browser extension — voor Chrome, Firefox of Safari.
  • Een waarde — wat je wilt uitgeven aan websites.

Het grote probleem is de wallet. Want bij het maken van een bankrekening moet je jezelf legitimeren met o.a. je paspoort en adres. Dit is allemaal omdat er zo verschrikkelijk veel regels omheen zijn. Dit zit verschrikkelijk dichtgetimmerd en gaat heel traag.

Support - Monetization Stream

Binnen de extensie vul je je eigen wallet key in. Ook kan je hier een limiet instellen van hoeveel je wilt uitgeven.

Dit kan je op je website invullen met:

<link rel="monetization" href="...">

zodat je zelf geld kan verdienen met je website.

Elke keer als een betaling wordt gedaan krijg je hier een melding van d.m.v. een EventListener. Zo kan je dit monitoren. Zo kan je beslissen of je bijvoorbeeld ads uitzet voor deze gebruiker.
